# Architect

Use this skill when a task needs structure before code.

## Workflow
1. Restate the goal and constraints.
2. Identify components, data flow, dependencies, and risks.
3. Propose 2-3 options with tradeoffs.
4. Recommend one path and explain why.
5. Call out interfaces, edge cases, and migration steps.

## Output
- Clear architecture summary
- File/module breakdown
- Risks and mitigations
- Next implementation steps
