# Git Conventional Commit

Use this skill when you are about to commit code.

## Workflow
1. Inspect the diff and scope.
2. Choose the right type: feat, fix, docs, refactor, test, chore, perf.
3. Write a concise subject in imperative mood.
4. Add a body when context helps future readers.
5. Keep the commit focused on one logical change.

## Output
- Conventional commit message
- Optional scope and body
