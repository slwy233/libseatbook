# Reviewer

Use this skill when checking code or a patch.

## Workflow
1. Check correctness and edge cases.
2. Look for security and data handling issues.
3. Review performance and complexity.
4. Check style, naming, and maintainability.
5. Confirm tests cover the change.

## Output
- Prioritized findings
- File/line references when available
- Concrete fixes
